Introduction

The WEBHUB Agents API allows developers to access multiple AI models—like GPT-4, Gemini, Grok, and Venice—through a single unified endpoint. This simplifies multi-agent integrations and ensures secure key management.

Authentication

All requests must be authenticated using an API key obtained from your Developer Dashboard.

Authorization: Bearer YOUR_API_KEY

API Endpoints

Below are the core endpoints:

  • POST /api/agent/:agentId/run — Send a prompt to a selected AI agent
  • GET /api/agents — Retrieve all available agents
  • GET /api/status — Check server health

Example Request

curl -X POST https://webhub.ai/api/agent/gpt-4/run \
  -H "Authorization: Bearer sk-xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Write a short poem about coding."
  }'

Example Response

{
  "provider": "openai",
  "data": {
    "choices": [
      { "message": { "content": "Code and dream in lines of light..." } }
    ]
  }
}

Error Handling

Common error responses:

{
  "status": 401,
  "message": "Invalid or missing API key."
}

Best Practices

  • Always cache non-sensitive requests to reduce latency.
  • Use the POST method for longer prompts or files.
  • Limit concurrent calls to avoid rate limit errors.

Rate Limits

Free Tier: 100 requests/day
Pro Tier: 10,000 requests/day
Enterprise: Custom limit with dedicated bandwidth

FAQ

Can I use multiple agents in one call?

Yes, using the multi-agent flag allows you to query several models simultaneously.

Where do I get my API key?

After signing up and verifying your account, navigate to the API Dashboard → Keys section.
